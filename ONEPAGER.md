# BriefDev — 한국 개발자를 위한 매일 5분 AI 트렌드 큐레이션

> **출퇴근 5분, 카드 5장.**
> Claude Code · Cursor · 새 모델 — 진짜 내 일에 영향 줄 5개만 골라드립니다.

---

## 🎯 왜 이 도메인을 선택했는가

**큐레이션 = 정보 격차가 곧 경쟁력 격차가 되는 영역**

AI 사주/이미지 생성 영역은 "한 번 써보고 끝"인 일회성 가치가 강합니다. 반면 큐레이션은 **매일 열어야 하는 가치**, 즉 리텐션이 본질이라 PE의 가설 검증력을 가장 잘 드러낼 수 있는 도메인이라고 판단했습니다.

특히 **"AI 정보 큐레이션"** 은 본인이 매일 필요로 하는 영역이고, 시장에 한국어 솔루션이 비어있어 검증 가능성과 차별점이 모두 명확했습니다.

---

## 🚨 기존 서비스가 풀지 못한 문제

### 타겟: AI 도구를 적극 사용하는 한국 IT 직장인 (3-8년차 개발자)

### 페르소나: 본인 (Brady, 모바일 개발자, AI 코딩 도구 적극 사용)

### 본인이 직접 겪은 문제 (1차 자료)
> *"AI 발전이 너무 빨라서 뭐가 나온지도 몰랐다.  
> 새 기능을 한참 후에 알게 되면 '진작 알았으면 더 빨리 만들었을 텐데' 하는 후회가 매주 반복된다."*

### 현재 정보 습득 경로의 한계

| 경로 | 한계 |
|------|------|
| **유튜브 구독** | 15분 영상 = 출퇴근 5분에 소화 불가 / 검색 불가 / 알고리즘 의존 |
| **영문 X(Twitter)** | 시간 부족 / 신호 대비 노이즈 비율 극악 |
| **한국 미디어 번역** | 1주일 이상 지연 (이미 늦음) |
| **뉴스레터 (미라클레터/어피티)** | 일반 시사 위주, 개발 직접 도움 X / 메일함에 묻힘 |
| **HackerNews 직접** | 영어 부담 / 30개 중 내게 의미 있는 5개를 찾는 데 시간 소모 |

### 한 줄 요약
> **"정보는 넘쳐나는데, 한국 개발자 맥락에서 '왜 내가 이걸 봐야 하는가'를 알려주는 곳이 없다."**

## 💡 가설과 핵심 기능

### 가설
> **만약** 카드 한 장에 *"왜 이게 한국 개발자에게 중요한가"* **한 줄을 붙이면**,  
> AI 도구를 쓰는 직장인은 **매일 1회 이상 앱을 열 것**이다.  
> **왜냐하면** 이들이 원하는 건 **정보 전달이 아니라 맥락 이해**이기 때문.

### 핵심 기능 (3개)

#### 1. 매일 아침 5장의 카드 (07:00 자동 갱신)
- **카테고리** (🔥 오늘의 1픽 / 📦 모델·도구 업데이트 / 💡 실전 팁 / 📚 깊이 읽을 거리 / 🎯 한국 화제)
- **3줄 요약** (한국어, 핵심만)
- **🎯 왜 한국 개발자에게 중요한가** (1줄 — **이게 진짜 차별점**)
- **원문 링크**

#### 2. 모바일 퍼스트 카드 스와이프 UI (PWA)
- 인스타 스토리처럼 좌우 스와이프 → 5초에 한 장
- 출퇴근 지하철 한 손 사용 최적화
- 홈 화면 추가로 앱처럼 사용

#### 3. "동료에게 1탭 공유"
- 슬랙/카톡/링크 복사
- 큐레이션의 본질은 **"이거 봤어?" 소통의 트리거**
- 자연스러운 바이럴 = CAC 0원 가설

### 의도적으로 만들지 않은 것 (PE의 우선순위 판단)

| 안 만든 것 | 이유 |
|-----------|------|
| 회원가입/마이페이지 | 매직링크 1단계 인증으로 갈음, 가설 검증에 무관 |
| 카테고리 커스터마이징 | 가설 검증 1단계는 "5개 카드의 가치"를 먼저 증명 |
| 결제/Pro 플랜 | Phase 2 가설, 지금은 검증 우선 |
| 푸시 알림 | PWA Web Push는 효과 대비 시간 소모 큼 |
| 실시간 알림 | 큐레이션의 가치는 "정해진 시간에 정해진 양" |

> **본인 판단**: 7일 안에 "10개 기능 다 만들기"는 풀스택 개발자의 일. PE는 **"가설 검증에 필요한 1개에 모든 디테일"** 을 쏟는 사람이라고 판단.

## 💰 수익 모델 가설

### Phase 1 (지금~1개월): 무료, 데이터 수집
**의도적으로 무료**. 가설 검증이 우선. 어떤 카테고리가 리텐션을 만드는가, 어떤 표현이 공유를 유발하는가를 먼저 학습.

### Phase 2 (1~3개월): B2C Freemium
- **Free**: 매일 5장
- **Pro ₩4,900/월**: 분야별 큐레이션 (백엔드/모바일/AI 리서치) + 음성 듣기 + 백넘버 검색 + 광고 제거
- 벤치마크: Readwise ($8/월), 더밀크(₩9,900/월)

### Phase 3 (3~6개월) ⭐ 진짜 베팅: B2B SaaS
**"팀 도메인 맞춤 AI 트렌드 다이제스트"**
- 핀테크/헬스테크/커머스 등 회사 도메인을 학습
- 매일 아침 슬랙으로 자동 전송
- 가격: 팀당 ₩99,000/월 (10명 미만), ₩490,000/월 (50명 미만)
- 100팀 → 월 매출 ₩990만~₩4,900만

**왜 B2B인가**:  
개인은 ₩4,900을 망설이지만, 회사는 "팀 정보 격차 해소" 명분에 ₩99,000을 즉결정합니다. 의사결정자가 다릅니다.

## 🛠 기술 스택 + AI 도구 활용

### 스택 (채용공고 우대 기술 매칭)
- **Frontend**: Next.js 14 (App Router) + TypeScript + Tailwind + shadcn/ui
- **Backend**: Next.js API Routes + Supabase (Auth + Postgres)
- **LLM**: Google Gemini 1.5 Flash (무료 티어, 1M 컨텍스트)
- **Cron**: Vercel Cron (매일 새벽 5시 자동 큐레이션)
- **Deploy**: Vercel

### AI 도구 활용 (Claude Code 하네스)
- **CLAUDE.md** — 도메인 용어/컨벤션/금지사항으로 일관된 코드 생성
- **prompts/curate-v1~v3.md** — 큐레이션 프롬프트 버전 관리 (A/B 테스트)
- **/pdca 워크플로우** — Plan-Do-Check 사이클 자동화
- **결과**: 3일 안에 1인 빌드 (전통 팀이 2주 걸릴 일)

> 자세한 활용 사례는 README.md 참고
