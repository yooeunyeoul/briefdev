# BriefDev — 한국 개발자를 위한 매일 5분 AI 트렌드 큐레이션

> **출퇴근 5분, 카드 5장.**
> Claude Code · Cursor · 새 모델 — 진짜 내 일에 영향 줄 5개만 골라드립니다.

---

## 🎯 왜 이 도메인을 선택했는가

**큐레이션 = 정보 격차가 곧 경쟁력 격차가 되는 영역**

AI 사주/이미지 생성 영역은 "한 번 써보고 끝"인 일회성 가치가 강합니다. 반면 큐레이션은 **매일 열어야 하는 가치**, 즉 리텐션이 본질이라 PE의 가설 검증력을 가장 잘 드러낼 수 있는 도메인이라고 판단했습니다.

특히 **"AI 정보 큐레이션"** 은 본인이 매일 필요로 하는 영역이고, 시장에 한국어 솔루션이 비어있어 검증 가능성과 차별점이 모두 명확했습니다.

---

## 🚨 기존 서비스가 풀지 못한 문제

### 타겟: AI 도구를 적극 사용하는 한국 IT 직장인 (3-8년차 개발자)

### 페르소나: 본인 (Brady, 모바일 개발자, AI 코딩 도구 적극 사용)

### 본인이 직접 겪은 문제 (1차 자료)
> *"AI 발전이 너무 빨라서 뭐가 나온지도 몰랐다.  
> 새 기능을 한참 후에 알게 되면 '진작 알았으면 더 빨리 만들었을 텐데' 하는 후회가 매주 반복된다."*

### 현재 정보 습득 경로의 한계

| 경로 | 한계 |
|------|------|
| **유튜브 구독** | 15분 영상 = 출퇴근 5분에 소화 불가 / 검색 불가 / 알고리즘 의존 |
| **영문 X(Twitter)** | 시간 부족 / 신호 대비 노이즈 비율 극악 |
| **한국 미디어 번역** | 1주일 이상 지연 (이미 늦음) |
| **뉴스레터 (미라클레터/어피티)** | 일반 시사 위주, 개발 직접 도움 X / 메일함에 묻힘 |
| **HackerNews 직접** | 영어 부담 / 30개 중 내게 의미 있는 5개를 찾는 데 시간 소모 |

### 한 줄 요약
> **"정보는 넘쳐나는데, 한국 개발자 맥락에서 '왜 내가 이걸 봐야 하는가'를 알려주는 곳이 없다."**

## 💡 가설과 핵심 기능

### 가설
> **만약** 카드 한 장에 *"~을 하는 개발자에게 — 어떤 가치"* **한 줄을 붙이면**,
> AI 도구를 쓰는 직장인은 **매일 1회 이상 앱을 열 것**이다.
> **왜냐하면** 이들이 원하는 건 **정보 전달이 아니라 "내 케이스인지 5초 안에 판단하기"** 이기 때문.

> **v1~v5 진화 과정에서 발견**: "왜 한국 개발자에게 중요한가" 추상 표현은 약했고,
> "**~을 하는 개발자에게 — 가치**" 형태가 사용자 자기 매칭을 가장 빠르게 만들었음.
> 5번의 production iteration으로 검증.

### 핵심 기능 (3개)

#### 1. 매일 아침 1~5장의 카드 (05:00 KST 자동 갱신)
- **카테고리** (🔥 오늘의 1픽 / 📦 모델·도구 / 💡 실전 팁 / 📚 깊이 읽기 / 🎯 한국 화제)
- **3줄 요약** (본문 근거, 환각 방지)
- **🎯 누구에게 유용한가** ("~을 하는 개발자에게 — 가치" 형태)
- **원문 링크** (페이월 도메인 자동 제외)

> 일별 카드 수가 **가변**입니다 (1~5장). 적합도 8점 이상 글이 부족한 날엔 정직하게 적게 보여줍니다 — 5장을 강제로 채우는 건 큐레이션 신뢰를 깬다는 판단.

#### 2. 모바일 퍼스트 카드 스와이프 UI
- 좌우 스와이프 + 드래그 회전 + dwell-time 추적
- 인터랙티브 요소(`<a>`)는 포인터 캡처 skip → 링크 클릭 통과
- 다크 테마 강제, 카드 자동 높이 (긴 콘텐츠 안 잘림)

#### 3. "동료에게 1탭 공유"
- Web Share API (모바일 네이티브) / 슬랙 / 클립보드 복사
- 카드별 고유 URL `/c/[id]` + 동적 OG 이미지 (1200×630)
- 카톡/슬랙 붙여넣으면 카드 디자인 그대로 미리보기 → 자연 바이럴 = CAC 0원 가설

### 의도적으로 만들지 않은 것 (PE의 우선순위 판단)

| 안 만든 것 | 이유 |
|-----------|------|
| 회원가입/마이페이지/관리자 | 매직링크로 갈음, 가설 검증에 무관 |
| **강제 회원가입** | 로그인은 "기록 남기고 싶다면" 선택사항. 첫 가치 체험에 마찰 0 |
| 카테고리 커스터마이징 | Phase 1은 "5장의 가치" 자체를 검증해야 함 |
| 결제/Pro 플랜 | Phase 2 가설, 지금은 검증 우선 |
| 푸시 알림 | PWA Web Push는 효과 대비 시간 소모 큼 |
| **시간 단위 실시간 갱신** | "출퇴근 5분 다이제스트" 가설은 1일 1번이 본질. 자주 갱신하면 가설 자체가 바뀜 |
| **5장 강제 채우기** | 적합한 글 부족하면 정직하게 1~4장 — 신뢰가 양보다 우선 |

> **본인 판단**: 7일 안에 "10개 기능 다 만들기"는 풀스택 개발자의 일. PE는 **"가설 검증에 필요한 1개에 모든 디테일"** 을 쏟는 사람이라고 판단.

### 로그인은 왜 있는가 — 정직한 답

**현재 사용자 가치**: 거의 없음. 익명으로도 카드 5장을 그대로 볼 수 있음.

**그럼 왜 만들었나**: Phase 2의 Pro 가치 후보를 검증할 **인프라**.
- 즐겨찾기 / 메모 / 주간 다이제스트 메일 / 관심 분야 설정 — 이런 기능을 Phase 2에 도입할 때, 그때 인증 시스템을 처음 만들면 늦음.
- 매직링크 + RLS + zod 검증된 `/api/views` 는 **30분에 만들었지만, 운영하면서 유저별 데이터 모델을 미리 검증하는 게 Phase 2의 토대**가 됨.
- `user_views` 테이블에 dwell time + 공유 여부가 쌓이면, "어떤 카테고리가 리텐션을 만드는가" 를 데이터로 답할 수 있음.

**왜 강제하지 않는가**: **첫 가치 체험 (5장 카드)에 마찰 0이 최우선**. "기록 남기고 싶다면" 작은 텍스트 링크 한 줄로만 노출. 평가자/사용자가 가입 없이 즉시 가치 체험 가능.

> 결제와 마찬가지로, 로그인도 **현재 사용자 가치 < 미래 가설 검증 인프라**. 의도적으로 미니멀하게 둠.

## 🪞 솔직한 한계 — 우리가 푼 것 vs 못 푼 것

PDF 평가 기준이 *"기술적 화려함 < 사용자에게 의미 있는 경험을 만드는 판단"* 이라, **약점 인지 자체를 정직하게 적습니다.**

### ✅ 이 제품이 푸는 것 (강한 차별)
- **정보 양 절제** — 하루 5장. GeekNews/HN의 정보 폭주와 다른 시나리오.
- **5초 자기 매칭** — `"~을 하는 개발자에게 — 가치"` 형태. 다른 큐레이션엔 없음.
- **환각 방어** — Jina Reader로 본문 추출 후 LLM에 → "120/110개" 같은 실제 수치 인용.
- **모바일 카드 스와이프 UX** — 출퇴근 한 손 사용 최적화.
- **출처 다양성** — 8개 소스에 글로벌 + 한국 균형.

### ⚠️ 여전히 못 푸는 것 (정직)
- **영어 부담 본질**: 카드 요약은 한국어지만 *"원문 읽기"* 클릭하면 대부분 영문 페이지. 페르소나의 핵심 문제(영어 부담)의 절반만 해결됨.  
  → Phase 2: 한국어 깊이 해설 / 본문 자체 한국어 변환.
- **시간 단위 실시간성**: 1일 1회 갱신. 모델 출시 직후 1시간 안에는 잡히지 않음.  
  → 의도적 트레이드오프 ("출퇴근 5분" 가설은 1일 1회가 본질).
- **가설 자체 검증 데이터 0**: 사용자 0명. D1 리텐션 데이터 없음.  
  → 인프라(`user_views`)는 깔렸음. 베이스라인 측정 후 가설 정량 갱신.
- **GeekNews와 컨텐츠 중복 가능성**: AI 태그 필터한 GeekNews 풀과 일부 겹칠 수 있음 (정확한 중복률 미측정 — 베이스라인 후 측정).  
  → 진짜 차별은 "콘텐츠"가 아니라 **"포맷 + 시나리오 + whyMatters 표현"**.

### 정직한 결론
> **"문제를 완전히 풀었다"가 아니라 "풀 수 있는 가설로 만들고 검증 인프라를 깔았다"** 가 정확.
>
> 평가자가 PDF에 명시한 *"문제를 발견하고, 가설을 세우고, 작은 제품으로 빠르게 검증하는 전 과정"* 의 **과정 자체가 산출물**.

## 📊 성공 지표

### 핵심 가설 검증 지표

> **원칙**: 모든 수치는 **출처 페이지를 직접 fetch해 verbatim 인용 확인**. 데이터가 직접 없거나 카테고리가 다른 항목은 *"베이스라인 후 결정"* 으로 명시. 추론·합성·N배 보정 없음.

#### 메트릭 우선순위 (제품 특성에 맞춤)

> 우리 제품은 **하루 5장 카드 컷오프**. 컨텐츠 양 자체가 제한적이라 *세션 시간이 길수록 좋음* 이 성립하지 않음 (가설 *"5분 다이제스트"* 와 충돌). 따라서:
>
> 1. **🥇 1순위: 리텐션 (D-N)** — *"매일 열어야 가치"* 가설의 직접 검증
> 2. **🥈 2순위: 카드 완독률** — 오늘 카드의 *콘텐츠 품질* 시그널
> 3. **측정만 (목표 없음): 세션 시간** — bounded 제품엔 *length = success* 부적합

#### 산업 anchor 데이터 (직접 fetch 검증 완료)

| 데이터 | 직접 인용 (verbatim) | 출처 |
|--------|---------------------|------|
| News 앱 D30 retention (Q3 2024) | *"news apps were reported in the third quarter of 2024 to have a retention rate of almost 10 percent"* (D30) | [Statista](https://www.statista.com/statistics/259329/ios-and-android-app-user-retention-rate/) |
| 글 완독률 (650K 페이지뷰 분석) | *"Only 20% of readers finished our articles. The average visitor would only read 25% of an article."* | [BDOW (formerly Sumo)](https://bdow.com/stories/how-many-visitors-read-article/) |

#### 1차 목표 (위 데이터에 anchor)

| 우선순위 | 지표 | 1차 목표 | 직접 인용 anchor |
|----------|------|---------|------------------|
| 🥇 핵심 | **D30 리텐션** | **10%+** | Statista News 앱 D30 *"almost 10%"* 동등 수준 도달 |
| 🥈 보조 | **카드 완독률** | **25%+** | BDOW: *"average visitor would only read 25% of an article"*. *카드 vs 글 형식 차이는 베이스라인 후 보정.* |
| ⏱️ 측정만 | **세션 시간** | **목표 없음** | bounded 5장 컷오프 제품엔 *"길수록 좋음"* 부적합. 측정은 하되 *적정 범위* 는 베이스라인 후 결정 |
| 📉 측정 후 | **D1 / D7 리텐션** | 베이스라인 후 결정 | 검증 가능한 News 앱 D1/D7 직접 데이터 없음 (Statista는 D30만) |
| 🔁 측정 후 | **공유율** | 베이스라인 후 결정 | 검증 가능한 모바일 앱 share rate 직접 데이터 없음 |

#### 실패 시그널 (직접 인용 anchor에 매칭)

| 실패 임계 | 직접 인용 anchor |
|----------|-----------------|
| ❌ **D30 < 10%** | Statista News 앱 D30 평균 *"almost 10%"* 미달 = News 카테고리 평균 이하 ⇒ *"매일 열어야 가치"* 가설 1차 부정 |
| ❌ **카드 완독률 < 20%** | BDOW 650K 분석: *"Only 20% of readers finished our articles"* 미달 ⇒ 카드 형식이 글보다 *나쁨* |
| ⚠️ **D1 / D7 / 공유율 / 세션 임계** | 검증 가능한 *floor* 데이터 없음 → **모두 베이스라인 후 결정** |

→ 위 *데이터 검증된 임계 2개 (D30·완독률)* 중 **하나 이상** 30일 내 해당하면 *현재 가설 (whyMatters → 5초 매칭 → 매일 열기)* 1차 갱신.

#### 한국 시장 정성적 컨텍스트 (보정 *근거 아님*)

> ⚠️ 아래는 *"한국 시장이 잘 받을 수도 있다"* 는 **정성적 시그널**. **글로벌 벤치마크에 N배 곱하는 정량 보정 근거 아님**. 1차 목표는 *글로벌 데이터에만 anchor*.

| 사실 | 출처 / 검증 상태 |
|------|------------------|
| 한국인 하루 평균 **스마트폰/PC 이용 5시간** (방송통신위원회 조사) | [Medifonews 2024](https://www.medifonews.com/news/article.html?no=184939) — verbatim *"한국인의 하루 평균 스마트폰/PC 이용 시간은 약 5시간이다"* ✅ 직접 fetch 확인 |
| 글로벌 모바일 사용자 1인당 하루 **약 3.5시간** | [Sensor Tower 2025](https://sensortower.com/ko/blog/state-of-mobile-2025-KR) — verbatim *"모바일 사용자 한 명당 하루에 약 3.5시간을 사용"* ✅ 직접 fetch 확인 |
| 한국 News 앱 잔존율 (전체 카테고리 1위) | [AppsFlyer Korea 2023](https://www.appsflyer.com/ko/resources/reports/app-retention-benchmarks/) — ⚠️ 페이지 fetch 500 error, verbatim 검증 미완 |
| 한국 소셜 사용자 ~19.7%가 앱에서 뉴스 소비 | [Adjust 2023 South Korea](https://www.adjust.com/blog/south-korea-in-three-charts-installs-sessions-and-retention-rates/) — ⚠️ rate limit 429, verbatim 검증 미완 |

> ⚠️ **한국 5h vs 글로벌 3.5h 직접 비교는 부정확**: 한국 5h는 *스마트폰 + PC* 합산, 글로벌 3.5h는 *모바일 only*. 카테고리 다름. 따라서 "한국이 N배 더 쓴다" 같은 정량 비교는 *불가*. *"한국 디지털 사용량이 많다"* 는 정성적 시그널로만 해석.
>
> **AppsFlyer / Adjust 한국 페이지는 fetch 실패** (500 / 429). 검색 스니펫에 의존된 수치라 **수치 정확성 미보장**. 베이스라인 후 한국 실측 데이터로 갱신 예정.

폐기 시 다음 가설 후보:
1. **카테고리 개인화** (모두에게 같은 5장 → 사용자별 가중치)
2. **시간대 다양화** (1일 1회 → 출근/점심/퇴근 3회)
3. **콘텐츠 깊이 변경** (요약 5장 → 1장 깊이 + 4장 요약)

### 측정 방법 (실제 구현된 것)
- **Vercel Analytics** — 페이지뷰, 세션, 디바이스 구성

## 💰 수익 모델 가설

### Phase 1 (지금~1개월): 무료, 데이터 수집
**의도적으로 무료**. 가설 검증이 우선. 어떤 카테고리가 리텐션을 만드는가, 어떤 표현이 공유를 유발하는가를 먼저 학습.

### Phase 2 (1~3개월): B2C Freemium
- **Free**: 매일 1~5장
- **Pro ₩4,900/월**: 분야별 큐레이션 (백엔드/모바일/AI 리서치) + 음성 듣기 + 백넘버 검색
- 글로벌 벤치마크 (직접 fetch 검증, 2026-04 기준):
  - **Readwise Full**: Annual **$5.59/월** / Monthly **$9.99/월** — [readwise.io/pricing](https://readwise.io/pricing) ✅ verbatim 확인
  - **더밀크**: **$25/월** (실리콘밸리 미디어) — [themiilk.com/payment](https://www.themiilk.com/payment) ✅ verbatim 확인
  - ⚠️ **Feedly Pro**: 페이지 로드 실패 (검증 미완) — 이전 검색 스니펫에 *$6.99/월* 있었으나 직접 확인 못함
- ₩4,900은 검증된 글로벌 비교군 ($5.59-$25) 보다 의도적으로 낮음 = "월 한 끼 점심값" 으로 진입 장벽 제거

### Phase 3 (3~6개월) ⭐ 진짜 베팅: B2B SaaS
**"팀 도메인 맞춤 AI 트렌드 다이제스트"**
- 핀테크/헬스테크/커머스 등 회사 도메인을 학습
- 매일 아침 슬랙으로 자동 전송
- 가격: 팀당 ₩99,000/월 (10명 미만), ₩490,000/월 (50명 미만)
- 100팀 → 월 매출 ₩990만~₩4,900만
- ⚠️ 글로벌 비교: Feedly Enterprise (검색 스니펫 *$1,600+/월부터* — Feedly 페이지 로드 실패로 직접 검증 미완)

**왜 B2B인가**:
개인은 ₩4,900을 망설이지만, 회사는 "팀 정보 격차 해소" 명분에 ₩99,000을 빠르게 결정함.
구매자(팀 리더)와 사용자(팀원)가 분리된 구조라 가격 저항이 다름.

## 🛠 기술 스택 + AI 도구 활용

### 스택 (채용공고 우대 기술 매칭)
- **Frontend**: Next.js 16 App Router + TypeScript strict + Tailwind v4
- **Backend**: Next.js API Routes + Supabase (Auth + Postgres + RLS)
- **LLM**: Google Gemini 2.5 Flash (무료 티어, 1M 컨텍스트, JSON Mode + 503 재시도)
- **본문 추출**: Jina Reader (`r.jina.ai`) — 환각 방지의 핵심
- **소스 (8개)**:
  - 글로벌 AI: HackerNews + Hugging Face Blog + Simon Willison + The Verge AI + Google AI Blog + GitHub Trending
  - 한국 개발자 커뮤니티: GeekNews + 토스 기술 블로그 + 카카오 기술 블로그
- **Cron**: Vercel Cron (매일 05:00 KST)
- **Deploy**: Vercel

### AI 도구 활용 (Claude Code 하네스)
- **CLAUDE.md** — 도메인 용어/컨벤션/금지사항으로 일관된 코드 생성
- **prompts/curate-v1~v5.md** — 큐레이션 프롬프트 버전 관리 (5번 진화)
- **/pdca 워크플로우** — Plan-Do-Check 사이클 자동화
- **결과**: 3일 안에 1인 빌드 — Plan/Design 문서 → 실제 코드까지의 사이클 시간을 *체감상* 단축 (정확한 측정값 없음)
